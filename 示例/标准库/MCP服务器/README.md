# MCP 服务器示例

本目录包含 Qi 语言 MCP (Model Context Protocol) 服务器的使用示例。

## 什么是 MCP？

MCP (Model Context Protocol) 是一个开放协议标准，用于 AI 应用与外部数据源和工具的集成。通过 MCP 服务器，可以：

- **注册工具 (Tools)**: 提供可执行的函数功能
- **管理资源 (Resources)**: 提供可访问的数据资源
- **定义提示 (Prompts)**: 提供预配置的提示模板

## API 参考

### 导入模块
```qi
包 主程序;

导入 标准库.MCP服务器;
```

### 创建服务器
```qi
变量 服务器ID: 整数 = MCP服务器.创建服务器(
    "服务器名称",
    "版本号",
    "描述"
);
```

### 注册工具
```qi
MCP服务器.注册工具(
    服务器ID,
    "工具名称",
    "工具描述"
);
```

### 注册资源
```qi
MCP服务器.注册资源(
    服务器ID,
    "资源URI",
    "资源名称",
    "资源描述",
    资源类型  // 0=文本, 1=二进制, 2=JSON
);
```

### 注册提示
```qi
MCP服务器.注册提示(
    服务器ID,
    "提示名称",
    "提示描述",
    "提示模板内容，使用{变量名}作为占位符"
);
```

### 启动/停止服务器
```qi
MCP服务器.启动服务器(服务器ID);
MCP服务器.停止服务器(服务器ID);
```

### 查询信息
```qi
// 获取服务器信息 (JSON)
```

### 清理资源
```qi

```

## 技术细节

### 协议版本
当前实现基于 MCP 协议版本 **2025-06-18**。

### 数据格式
所有列表查询函数返回 JSON 格式的字符串，可以使用 JSON 标准库进行解析。

### 资源类型
- **0**: 文本资源 (text)
- **1**: 二进制资源 (blob)
- **2**: JSON 资源 (json)

### 返回值约定
- 成功操作返回 `0` 或正整数 ID
- 失败操作返回 `-1` 或 `NULL`
- 运行状态查询返回 `1` (运行) 或 `0` (未运行)

## 注意事项

1. **服务器生命周期**: 使用完服务器后应调用 `销毁服务器()` 释放资源
2. **字符串释放**: 从 FFI 返回的字符串由 Qi 运行时自动管理
3. **工具执行**: 当前版本工具执行功能需要在 Rust 层设置执行函数
4. **并发访问**: 服务器池使用 Mutex 保护，支持并发访问
